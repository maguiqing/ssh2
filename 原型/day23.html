5、对象的创建方式  --- 单个对象的创建

var obj = { 
    "name" : "xm",
    "study" : function(){
        console.log(xxx)
    }
}
或
var obj = new Object();
obj.name = "xm";
obj.running = function(){ ... }
缺点 ： 一次只能创建一个对象 

6、对象创建方式之--- 工厂模式  --- 批量创建多个同类的对象

优点 ： 可以批量创建多个同类的对象，解决了多个同类对象创建时代码重复的问题
缺点 ： 对象的创建就是普通函数的调用 ， 一般一个对象是通过new关键字创建的  ， 不符合对象创建的规范 
       不能确定某个对象属于哪一个类（构造函数）   

7、对象创建方式之 -- 构造函数

系统提供构造函数 ： Array  Date  String Number RegExp   Promise  。。。。
自定义构造函数：
    1、为了和普通函数区分  一般构造函数的名大驼峰  
    2、构造函数中的this 指向 new 出来的对象
    3、写在构造函数内部的属性 叫做 实例属性   写在构造函数内部的方法  叫做  实例方法
优点 ： 
    对象创建是通过new关键字创建的  符合对象的创建规范 
    能够确定某个对象属于哪一个类

缺点 ：  同类对象的方法会重复创建   占不同内存空间

8、对象创建方式之 ---  原型模式

1、创建一个构造函数
2、将对象的属性和方法写在构造函数的外面   这样的属性叫做原型属性 ，方法叫做原型方法

优点 ： 解决了多个同类对象创建时  方法被重建的问题   方法共享一块内存空间
缺点 ： 多个对象的属性值相同        
9、对象创建方式之 -- 混合

将属性写成实例属性
将方法写成原型方法
